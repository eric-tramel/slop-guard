Install / Run

Catch slop before it lands in your docs.

slop-guard is a rule-based prose linter for formulaic AI writing. It scores text from 0 to 100, points to the exact spans that dragged the score down, and returns direct advice for the rewrite.

No LLM judge. No API calls. No cost.

Run Once

uvx --from slop-guard sg README.md

Install The CLI

uv tool install slop-guard

Add To Codex

codex mcp add slop-guard -- uvx slop-guard

Add To Claude Code

claude mcp add slop-guard -- uvx slop-guard

What It Returns

Compiled rules, visible reasons, repeatable output.

The analyzer walks words, phrases, sentence patterns, formatting, and density signals. Instead of a vibe-based verdict, you get a score, the triggered rule hits, surrounding context, and concrete advice for each match.

Score

0-100 banding

Clean, light, moderate, heavy, or saturated. Useful in a terminal, inside an agent, or as a CI threshold.

Matches

Exact spans

Each violation includes the matched text, surrounding context, and penalty that pushed the draft down.

Advice

Rewrite prompts

The output tells you what to cut or replace so you can move from filler to specific claims fast.

Ways To Use It

Scroll through the same lint pass in different workflows.

Start inside an agent, drop to the CLI, turn it into a publishing gate, or fit the rules to your own corpus. The preview on the right follows the card in view.

01

Inside Codex or Claude

Add slop-guard as an MCP server and let the agent call check_slop or check_slop_file when it drafts docs, release notes, or status updates.

02

At the terminal

Run sg on files, inline prose, or stdin. Good for README edits, landing-page copy, changelogs, and generated text.

03

As a quality gate

Use thresholds in CI so doc quality has an enforced floor. Low-scoring prose fails before it ships.

04

Fit it to your own style

Train a JSONL rule config from positive and negative corpora, then point both the CLI and MCP server at the fitted rules.
